Introduction

Claude Code is Anthropic's agentic coding tool designed to help developers turn ideas into code faster. It works directly in your terminal, can edit files, run commands, create commits, and integrates seamlessly with your existing development workflow.

Installation & Setup

Prerequisites

  • Terminal or command prompt
  • Node.js 18 or newer (recommended)
  • A code project to work with

Installation Methods

Method 1: NPM (Recommended)

$ npm install -g @anthropic-ai/claude-code

Method 2: Native Install (Beta)

For macOS, Linux, WSL:

$ curl -fsSL claude.ai/install.sh | bash

For Windows PowerShell:

PS> irm https://claude.ai/install.ps1 | iex

First Time Setup

$ cd /path/to/your/project
$ claude
✓ Authenticated with Anthropic
✓ Ready to assist with your code!

CLI Reference

Basic Commands

Command Description
claude Start interactive REPL
claude "prompt" Start with initial prompt
claude -p "query" Query and exit (SDK mode)
claude -c Continue most recent conversation
claude -r "id" Resume specific session
claude update Update Claude Code
claude mcp Configure MCP servers

Command Line Flags

Flag Description
--add-dir Add additional working directories
--allowedTools Specify allowed tools (comma-separated)
--disallowedTools Specify disallowed tools
-p, --print Print response without interactive mode
--verbose Enable detailed logging
--max-turns Limit agentic turns (default: 10)
--model Set session model
--permission-mode Set permission mode (strict/relaxed)

Memory Management

CLAUDE.md System

Claude Code uses a hierarchical memory system with four levels:

Memory Hierarchy

  1. Enterprise Policy - /etc/claude/CLAUDE.md
  2. Project Memory - .claude/CLAUDE.md
  3. User Memory - ~/.claude/CLAUDE.md
  4. Local Project - ./CLAUDE.md (deprecated)

Best Practices for CLAUDE.md

# Project Overview
Brief description of the project and its purpose

## Architecture
- Key components and their relationships
- Technology stack
- Design patterns used

## Development Guidelines
- Code style preferences
- Testing requirements
- Commit message format

## Important Context
- Business rules
- Edge cases to consider
- Performance requirements

## Common Tasks
- Build commands
- Test commands
- Deployment process

Security & Privacy

🔒 Security Model

  • Read-only by default
  • Explicit approval for writes
  • Scoped to working directory
  • Context-aware threat detection
  • Input sanitization
  • Command blocklist

Best Practices

Review Before Approval

> Claude wants to run: rm -rf /
> Approve? [y/N] N

Use Virtual Environments

docker run -it --rm \
  -v $(pwd):/app \
  node:latest claude

Limit Permissions

claude \
  --disallowedTools bash,write

Troubleshooting

Common Issues and Solutions

Installation Problems

# npm install fails
nvm use 18
npm install -g @anthropic-ai/claude-code

# Command not found
export PATH="$PATH:$(npm root -g)/bin"

Permission Errors

# Cannot edit files
/permissions
claude config set permissionMode relaxed

# Command blocked
claude --dangerously-skip-permissions

Performance Issues

# Slow responses
/compact
/clear

# Token limit reached
/cost
# Consider upgrading plan

Connection Problems

# Authentication failed
/logout
/login

# API errors
/status
/doctor
