30-Day Django Series - Day 1

Understanding Django: MVT Architecture and a Mini Blog Project

December 26, 2024 • 8 min read • Django, Python, Tutorial

Django is a high-level Python web framework that enables rapid development with clean, pragmatic design. On Day 1 of my Django interview prep journey, I focused on understanding what Django is, the MVT architecture, and creating a mini blog project to solidify the concepts.

What is Django?

Django is a "batteries-included" framework. It comes with built-in features like ORM, authentication, admin interface, templates, and security tools. This makes it perfect for developers who want to rapidly build robust web applications without reinventing the wheel.

MVT vs MVC

Django follows the MVT (Model–View–Template) pattern, which is a variation of the classic MVC (Model–View–Controller):

Aspect MVC Django MVT
Model Handles data & DB logic Same as MVC
View Handles UI rendering Template handles UI rendering
Controller Handles business logic + routing View handles logic (acts as Controller)
Routing Usually in Controller Handled by Django's URL dispatcher

Key takeaway: In Django, the "Controller" is hidden inside the framework, making it easier to focus on writing views and templates.

Mini Project: My First Django Blog

To put theory into practice, I built a simple blog app.

Step 1: Create Project and App

django-admin startproject blogproject
cd blogproject
python manage.py startapp blog

Step 2: Define Model

from django.db import models

class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.title
  • Run migrations: makemigrations + migrate

Step 3: Add View

from django.shortcuts import render
from .models import Post

def home(request):
    posts = Post.objects.all()
    return render(request, "blog/home.html", {"posts": posts})

Step 4: Configure URL

from django.urls import path
from . import views

urlpatterns = [
    path("", views.home, name="home"),
]

Step 5: Create Template

<!DOCTYPE html>
<html>
<head>
    <title>Django MVT Example</title>
</head>
<body>
    <h1>My Blog</h1>
    {% for post in posts %}
        <h2>{{ post.title }}</h2>
        <p>{{ post.content }}</p>
        <small>{{ post.created_at }}</small>
    {% empty %}
        <p>No posts yet!</p>
    {% endfor %}
</body>
</html>

✅ Result: The homepage dynamically shows all posts stored in the database.

Unit Testing

I also learned how to write basic unit tests to ensure the app works as expected:

from django.test import TestCase
from .models import Post

class PostModelTest(TestCase):
    def setUp(self):
        self.post = Post.objects.create(title="Test Post", content="Hello Django")

    def test_post_creation(self):
        self.assertEqual(self.post.title, "Test Post")
        self.assertEqual(str(self.post), "Test Post")
  • Run tests: python manage.py test
  • Tests verify the model creation and __str__ method.

Key Learnings

  1. Django uses MVT instead of MVC, making development more streamlined.
  2. Models handle data, Views handle logic, Templates handle presentation.
  3. The Django admin interface is powerful for quickly managing data.
  4. Unit testing is important even for small projects—it ensures your code works and builds confidence.
  5. Following best practices like separating logic and presentation makes your app scalable.

Best Practices

  • Keep business logic in Views, not Templates.
  • Use Template Inheritance to avoid duplicate HTML.
  • Name URLs clearly and use {% url 'home' %} in templates.
  • Write unit tests from the start.
